# مواصفة مشروع DUMLIS (Spec Kit)

## نظرة عامة
تطوير نظام DUMLIS من نسخة تجريبية إلى نظام إنتاج متكامل يدعم تعدد الكليات (Multi-Faculty) مع إكمال جميع الوحدات الوظيفية.

## المتطلبات الوظيفية (Functional Requirements)

### 1. عزل البيانات (Data Isolation)
- يجب أن يرى مدير الكلية (Faculty Admin) بيانات كليته فقط (طلاب، أقسام، كورسات، غرف).
- يجب أن يرى رئيس الجامعة (Super Admin) جميع البيانات مع إمكانية التبديل بين الكليات.
- يجب تصفية جميع استعلامات الـ API بناءً على `faculty_id`.

### 2. إدارة الطلاب المتقدمة
- إكمال ربط حقول البيانات (العسكرية، الطبية، العائلية).
- دعم عمليات الـ CRUD الكاملة (إضافة، تعديل، حذف، عرض).

### 3. نظام الامتحانات والمالية
- توليد أرقام الجلوس أوتوماتيكياً بناءً على سعة الغرف.
- تتبع مدفوعات الطلاب وحساب المديونيات.

## المتطلبات التقنية (Technical Requirements)
- **Backend**: FastAPI + SQLAlchemy (PostgreSQL).
- **Frontend**: React + TypeScript + Vite.
- **State Management**: React Context / Hooks.
- **Testing**: Playwright (E2E).

## الحدود (Out of Scope)
- نظام الحضور عبر البصمة (يؤجل للمرحلة القادمة).
- تكامل الدفع الإلكتروني (فيزا/ماستر كارد) - حالياً تتبع يدوي فقط.
