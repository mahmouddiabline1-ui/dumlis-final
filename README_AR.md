# 🎓 نظام DUMLIS - إدارة التعليم الجامعي
## DUMLIS - University Learning Management System

---

## 📌 ملخص النظام | System Overview

**DUMLIS** هو نظام إدارة أكاديمية شامل للجامعات، يوفر:

- 🎯 **إدارة الطلاب**: تسجيل، متابعة، تحديث البيانات الشاملة
- 📚 **إدارة المقررات**: إنشاء وجدولة وتقييم المقررات الدراسية
- 👥 **إدارة الموارد البشرية**: الأساتذة والموظفين والإداريين
- 📊 **الإحصائيات والتقارير**: تقارير مفصلة وحية عن الأداء الأكاديمي
- 💰 **إدارة المالية**: الرسوم والدفوعات والحسابات المالية
- 🏫 **إدارة الامتحانات**: جدولة الامتحانات وترتيب الجلوس
- 🔐 **أمان وعزل البيانات**: كل كلية ترى بيانتها فقط

---

## 🚀 البدء السريع
## Quick Start

### الطريقة السريعة (10 دقائق):
```bash
# 1. إعداد قاعدة البيانات
cd backend
python seed_multi_faculty.py

# 2. تشغيل الخادم (Terminal 1)
python -m uvicorn app.main:app --reload --port 8000

# 3. تشغيل الواجهة (Terminal 2)
cd frontend
npm run dev

# 4. افتح المتصفح
# http://localhost:5173
```

### حسابات الاختبار:
```
Super Admin:      admin / admin123
Faculty Admin:    fcai_admin / fcai123
Student Affairs:  student_affairs / sa123
```

## 🏗️ البنية المعمارية
## Architecture

### الخادم الخلفي | Backend
```
FastAPI + SQLAlchemy + PostgreSQL

📁 backend/
├── app/
│   ├── main.py          ← نقطة الدخول الرئيسية
│   ├── models.py        ← نماذج قاعدة البيانات
│   ├── database.py      ← إعدادات قاعدة البيانات
│   ├── schemas.py       ← Pydantic schemas
│   ├── auth.py          ← مصادقة وتفويض
│   └── routers/         ← جميع الـ API endpoints
│       ├── students.py
│       ├── courses.py
│       ├── rooms.py
│       ├── committees.py
│       └── ... (20+ routers)
├── seed_multi_faculty.py ← ملء البيانات الواقعية
└── requirements.txt
```

### الواجهة الأمامية | Frontend
```
React + TypeScript + Vite

📁 frontend/
├── api.ts               ← عميل API
├── lookupApi.ts         ← خدمة البحث الديناميكية
├── components/          ← مكونات React
│   ├── StudentDataManagement.tsx
│   ├── DbBackedPage.tsx
│   ├── CourseManagement.tsx
│   └── ... (40+ components)
├── data/                ← البيانات والتكوينات
│   ├── formOptions.ts   ← خيارات النماذج الديناميكية
│   ├── committeesData.ts ← خدمة الغرف واللجان
│   └── pageConfig.ts    ← تكوينات الصفحات
└── App.tsx
```

## 🚀 النشر
## Deployment

### متطلبات الإنتاج | Production Requirements
```
- PostgreSQL 12+ (بقاعدة بيانات مستقلة)
- Python 3.8+ (مع مكتبات محدثة)
- Node.js 14+ (لبناء الواجهة الأمامية)
- HTTPS (شهادة SSL)
- نسخ احتياطية تلقائية
- مراقبة 24/7
```

### خطوات النشر | Deployment Steps
1. **التحضير**:
   ```bash
   # نسخة احتياطية من قاعدة البيانات
   pg_dump dumlis > backup.sql
   ```

2. **التطبيق**:
   ```bash
   # تطبيق الهجرات
   alembic upgrade head
   ```

3. **البناء**:
   ```bash
   # بناء الواجهة الأمامية
   cd frontend && npm run build
   ```

4. **التشغيل**:
   ```bash
   # تشغيل الخادم
   gunicorn -w 4 -k uvicorn.workers.UvicornWorker app.main:app
   ```

## 🐛 استكشاف الأخطاء
## Troubleshooting

### المشاكل الشائعة | Common Issues

| المشكلة | الحل |
|--------|------|
| قاعدة البيانات فارغة | تشغيل `python seed_multi_faculty.py` |
| خطأ CORS | التحقق من `VITE_API_URL` في `.env.local` |
| القوائم المنسدلة فارغة | إعادة تحميل الصفحة أو حذف الذاكرة المؤقتة |
| فشل الدخول | التحقق من بيانات المستخدم في قاعدة البيانات |
| بطء الأداء | فحص استعلامات قاعدة البيانات والفهارس |
